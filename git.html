git cheatsheet

Resources Assignments git cheatsheet
  • git init
  • git status
  • git add <filename>
  • git rm <filename>
  • git reset <filename>
  • git commit
  • git diff
  • git log
  • git checkout: Change the workspace (i.e. the current files in the directory) to those located at the specified commit. In essence, moves "you" (a.k.a. HEAD) to the commit. If you add the -b flag, it will create a new branch with the specified name: git checkout -b <new_branch_name>.
  • git branch <branchname>
  • git remote
  • git push
  • git merge <branchname>
  • git fetch: Downloads objects and refs from another repository. Fetch refs from one or more other repositories, along with the objects necessary to complete their histories. The names of refs that are fetched, together with the object names they point at, are written to .git/FETCH_HEAD.
  • git pull <remote_repo> <remote_branch>: Runs git fetch and calls git merge to merge retrieved branch heads into current branch (can also use --rebase to run git rebase instead of git merge). Fetches and replays changes from remote branch since it has diverged from the local branch
